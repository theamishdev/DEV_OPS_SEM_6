# Day 6: Docker Interaction, Volumes & Networking

---

## 1. Container Interaction Commands

### 1. Execute Command Inside Container

docker exec -it <container_id> /bin/bash


- `-i` → Interactive mode
- `-t` → Terminal
- Used to access container shell

---

### 2. Run Container in Detached + Interactive Mode

docker run -dit ubuntu /bin/bash


- `-d` → Detached mode (runs in background)
- `-i` → Keep STDIN open
- `-t` → Allocate terminal

---

### 3. View Container Logs

docker logs <container_id>


- Displays logs generated by container
- Useful for debugging

---

### 4. Inspect Container Details

docker inspect <container_id>


- Provides detailed JSON output
- Includes:
  - IP address
  - Mounts
  - Configuration

---

### 5. View Container Stats

docker stats


- Shows live resource usage:
  - CPU
  - Memory
  - Network

---

### 6. List Running Processes in Container

docker top <container_id>


- Displays active processes inside container

---

## 2. Volume Commands

### What is a Volume?

A **Docker Volume** is used for **persistent data storage**.

- Data is stored outside container
- Survives container deletion

---

### 1. Create Volume

docker volume create <volume_name>


---

### 2. List Volumes

docker volume ls


---

### 3. Inspect Volume

docker volume inspect <volume_name>


- Shows:
  - Mount path
  - Usage details

---

### 4. Remove Volume

docker volume rm <volume_name>


---

## 3. Docker Network Commands

### What is Docker Networking?

Docker networking allows containers to **communicate with each other and external systems**.

---

### 1. List Networks

docker network ls


---

### 2. Create Network

docker network create <network_name>


---

### 3. Inspect Network

docker network inspect <network_name>


- Shows connected containers
- Displays IP allocation

---

### 4. Run Container in Custom Network (with -dit)

docker run -dit --name mycontainer --network <network_name> ubuntu


---

### 5. Connect Container to Network

docker network connect <network_name> <container_id>


---

### 6. Disconnect Container from Network

docker network disconnect <network_name> <container_id>


---

### 7. Remove Network

docker network rm <network_name>


---

## 4. Types of Docker Networks

| Network Type | Description |
|-------------|------------|
| Bridge | Default network for containers on same host |
| Host | Shares host network directly |
| None | No network access |
| Overlay | Used for multi-host communication |

---

## 5. Summary

- `-dit` is commonly used to run containers in **background with interactive terminal**
- `docker exec`, `logs`, `inspect`, `stats`, `top` → used for container interaction
- Volumes provide **persistent storage**
- Networks enable **container communication**
- Together, they form the backbone of **real-world Docker applications**
