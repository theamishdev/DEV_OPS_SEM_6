# Day 1: What is DevOps?

1. DevOps is a set of practices, culture, and tools that integrates software development (Dev) and IT operations (Ops) to build, test, release, and deploy software faster and more reliably.

Simple defination: "DevOps is a methodology that combines development and operations teams to automate and streamline software delivery."


Traditional model:
Development → Testing → Release → Operations

DevOps model:
Development ↺ Integration ↺ Testing ↺ Deployment ↺ Monitoring

It becomes a continuous cycle.



# Why Devops?

1. Collaboration: Developer and operations team work together throughout the lifecycle. 

Instead of blaming each other for failures:

Dev: "It works on my machine."
Ops: "But it crashes in production."

DevOps eliminates this conflict by making them share responsibility.


2. Automation: Many tasks are automated: * Code Building
                                         * Testing
                                         *Deployment
                                         *Infrastructure setup
                                         * Monitoring
    
Automation reduces: * Human error 
                    *deployment time
                    *Operational cost



3. Continuos Processes: DevOps promotes continuous practices such as:

* Continuous Integration (CI)

* Continuous Delivery (CD)

* Continuous Deployment

* Continuous Monitoring

This ensures software is always ready to release.
